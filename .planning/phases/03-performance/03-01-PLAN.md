In `src/lib/queries.ts`, refactor `getFlavorBySlug(slug)` to parallelize independent queries.

**Current sequential flow (6 awaits in series):**
1. Fetch flavor (await) -- MUST be first, others depend on flavor.id
2. Fetch ratingsRaw (await) -- depends on flavor.id
3. Fetch users by userIds (await) -- depends on ratingsRaw
4. Fetch likes by ratingIds (await) -- depends on ratingsRaw
5. Fetch myLikes (await) -- depends on ratingsRaw + currentUser
6. Fetch siblingFlavors (await) -- depends on flavor.product_id

**Optimized flow:**
Step 1: Fetch flavor (await) -- still first, no change
Step 2: Parallel batch -- `ratingsRaw` and `siblingFlavors` can run together (both depend only on `flavor`):
```typescript
const [{ data: ratingsRaw }, { data: siblingFlavors }] = await Promise.all([
  db.from('ratings').select('*').eq('flavor_id', flavor.id).order('created_at', { ascending: false }).limit(20),
  db.from('flavors').select('id, name, slug').eq('product_id', flavor.product_id).neq('id', flavor.id).order('name').limit(20),
])
```

Step 3: Get currentUser in parallel with nothing (or combine):
```typescript
const { data: { user: currentUser } } = await supabase.auth.getUser()
```
Actually, `getUser()` can be moved to run in parallel with Step 2 since it doesn't depend on flavor data. Add it to the Step 2 Promise.all:
```typescript
const [{ data: ratingsRaw }, { data: siblingFlavors }, { data: { user: currentUser } }] = await Promise.all([
  db.from('ratings')...,
  db.from('flavors')...,
  supabase.auth.getUser(),
])
```

Step 4: After ratingsRaw is available, build userIds and ratingIds, then parallel batch for users + likes + myLikes:
```typescript
const userIds = [...new Set(allRatings.map((r: any) => r.user_id))]
const ratingIds = allRatings.map((r: any) => r.id)

const [usersResult, likesResult, myLikesResult] = await Promise.all([
  userIds.length > 0 ? db.from('users').select('id, username, badge_tier, avatar_url').in('id', userIds) : Promise.resolve({ data: [] }),
  ratingIds.length > 0 ? db.from('review_likes').select('rating_id').in('rating_id', ratingIds) : Promise.resolve({ data: [] }),
  currentUser && ratingIds.length > 0 ? db.from('review_likes').select('rating_id').eq('user_id', currentUser.id).in('rating_id', ratingIds) : Promise.resolve({ data: [] }),
])
```

Keep the rest of the function (mapping, return statement) unchanged. The return shape MUST remain identical.

IMPORTANT: Guard all `.in()` calls with length > 0 checks (already done in current code, preserve this pattern).
