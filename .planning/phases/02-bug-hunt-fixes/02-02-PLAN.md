  <action>
The success page currently does:

```typescript
const { flavor } = data
const product = flavor.product
const brand = (product as any).brands
```

Then renders `{product.name}` on line 148 (unguarded) and `{brand?.name && ...}` (guarded). The risk is:

1. `product` can be null at runtime — the TypeScript cast `Product & { brands: Brand }` hides this. `product.name` would throw.
2. `brand` is already accessed with optional chaining (`brand?.name`) in the conditional, so that part is safe.

Make these two targeted changes:

**Change 1 — Add null-safe fallback for product** (line 36, right after `const product = flavor.product`):

Current:
```typescript
const { flavor } = data
const product = flavor.product
const brand = (product as any).brands
```

Change to:
```typescript
const { flavor } = data
const product = flavor.product as (typeof flavor.product & { brands?: { name: string } | null }) | null
const brand = (product as any)?.brands ?? null
```

**Change 2 — Guard `product.name` on line 148** (inside the brand/product subtitle paragraph):

Current:
```tsx
{brand?.name && (
  <p
    style={{
      color: 'var(--text-dim)',
      fontSize: '13px',
      margin: '4px 0 0',
      fontWeight: 600,
    }}
  >
    {brand.name} · {product.name}
  </p>
)}
```

Change to:
```tsx
{(brand?.name || product?.name) && (
  <p
    style={{
      color: 'var(--text-dim)',
      fontSize: '13px',
      margin: '4px 0 0',
      fontWeight: 600,
    }}
  >
    {brand?.name ? `${brand.name} · ` : ''}{product?.name ?? ''}
  </p>
)}
```

These are the only two changes. Do not alter any other logic, layout, or styling in the file.
