**Sticky submit button positioning (lines 487-501):**
The sticky submit currently uses `bottom: 0` and `zIndex: 10`. On mobile, this places it behind the bottom nav (z-index 50, height 64px). The layout.tsx `<main>` has bottom padding that pushes content above the nav, but the sticky element itself sticks to `bottom: 0` of the scroll container, which can overlap the nav visually.

Change the sticky submit div's style:
- Change `bottom: 0` to `bottom: 'calc(64px + env(safe-area-inset-bottom))'` -- this is a mobile-only concern but since the form is primarily used on mobile, apply it unconditionally. On desktop where BottomNav is hidden (`sm:hidden`), the extra spacing is acceptable since the form has `maxWidth: '600px'`.

Actually, the better approach: use a CSS media query approach via a wrapper. Since this is inline styles, instead add the bottom offset directly. Change the style from:
```js
bottom: 0,
```
to:
```js
bottom: 0,
```
Wait -- the layout.tsx main already has `pb-[calc(64px+env(safe-area-inset-bottom))]` which means the scroll container itself has bottom padding. When the user scrolls to the bottom, the sticky element at `bottom: 0` will be above the bottom nav because the padding pushes it up. The issue is only when NOT fully scrolled -- the sticky element may overlap the nav mid-scroll.

The correct fix: change the sticky container `bottom` from `0` to `0` but ensure the form container's `paddingBottom: '96px'` (line 205) is sufficient. Currently 96px > 64px, so the content scrolls past the nav. The real risk is the sticky element overlapping mid-scroll.

Fix: Change `zIndex: 10` to `zIndex: 45` so the sticky submit sits above the general content but below the bottom nav (z-index 50). This prevents visual overlap -- the nav will cover the submit when it's at bottom: 0, but the user can scroll past it. The paddingBottom: 96px on the form container ensures the submit scrolls above the nav at the bottom.

Additionally, to be safe and ensure the submit is always visible on mobile, change the `paddingBottom` of the form container (line 205) from `'96px'` to `'calc(96px + 64px + env(safe-area-inset-bottom))'`. This ensures the bottom of the form scrolls well past the bottom nav, making the sticky submit visible above it.

Summary of changes to RatingForm.tsx:
1. Line 205: Change `paddingBottom: '96px'` to `paddingBottom: 'calc(160px + env(safe-area-inset-bottom))'` (96 + 64 = 160px, accounts for bottom nav)
